Overview

Unit 4 introduced p5.js — a JavaScript library for creative coding. I built interactive sketches, learned game-loop architecture with setup() and draw(), and ultimately created a full original game. Below are three milestone tasks, the game tutorial I followed, and my final project.

Milestone 1 — Bouncing Ball

This was my first interactive p5.js sketch. A coloured ellipse bounces around the canvas, reversing direction each time it hits a wall. Clicking the ball changes its colour randomly and reverses its direction. I learned how draw() acts as a continuous loop, how to track position and speed with variables, and how to use mousePressed() with dist() for click detection.

let radius = 50;
let x = 200;
let y = 200;
let speedX = 2;
let speedY = 2;
let r = 208;
let g = 0;
let b = 120;

function setup() {
  createCanvas(600, 400);
}

function draw() {
  background(40);
  fill(r, g, b);
  noStroke();
  ellipse(x, y, radius * 2);

  x += speedX;
  y += speedY;

  if (x > width - radius || x < radius) speedX = -speedX;
  if (y > height - radius || y < radius) speedY = -speedY;
}

// Click the ball to change its colour and reverse direction
function mousePressed() {
  let d = dist(mouseX, mouseY, x, y);
  if (d < radius) {
    r = random(255);
    g = random(255);
    b = random(255);
    speedX = -speedX;
    speedY = -speedY;
  }
}

Milestone 2 — Clicker Game

I built a 30-second reflex game where the player clicks a moving circle as many times as possible before the timer runs out. Building this taught me how to manage game states (playing vs. game over), use setInterval() for a countdown, and reset all variables on restart. The score and timer are drawn to the canvas each frame using the text() function.

let target = { x: 200, y: 200, radius: 40 };
let score = 0;
let timeLeft = 30;
let gameOver = false;

function setup() {
  createCanvas(500, 400);
  textAlign(CENTER, CENTER);
  // Countdown: decrease timeLeft once per second
  setInterval(function() {
    if (!gameOver) {
      timeLeft--;
      if (timeLeft <= 0) gameOver = true;
    }
  }, 1000);
}

function draw() {
  background(30, 30, 50);
  fill(255);
  textSize(28);
  text("Click The Circle!", width / 2, 40);
  textSize(20);
  text("Score: " + score, width / 2, 70);
  text("Time: " + timeLeft, width - 80, 40);

  if (gameOver) {
    fill(255, 100, 100);
    textSize(40);
    text("GAME OVER", width / 2, height / 2);
    textSize(24);
    text("Final Score: " + score, width / 2, height / 2 + 50);
    text("Click to Restart", width / 2, height / 2 + 90);
    return;
  }

  fill(100, 200, 255);
  noStroke();
  ellipse(target.x, target.y, target.radius * 2);
}

function mousePressed() {
  if (gameOver) {           // restart on click after game over
    score = 0;
    timeLeft = 30;
    gameOver = false;
    return;
  }
  let d = dist(mouseX, mouseY, target.x, target.y);
  if (d < target.radius) {
    score++;
    target.x = random(target.radius, width - target.radius);
    target.y = random(100, height - target.radius);
  }
}

Milestone 3 — Minesweeper Grid

This milestone taught me how to use 2D arrays to represent a grid of cells. Each cell is an object that stores its position, whether it's a mine, whether it's been revealed, and whether it's been flagged. Left-clicking reveals a cell; right-clicking toggles a flag. Building this reinforced nested loops, object properties, and mouse-button detection with mouseButton === RIGHT.

let cols = 10;
let rows = 10;
let cellSize = 40;
let grid = [];

function setup() {
  createCanvas(cols * cellSize, rows * cellSize);
  document.oncontextmenu = function() { return false; }; // disable right-click menu

  for (let c = 0; c < cols; c++) {
    grid[c] = [];
    for (let r = 0; r < rows; r++) {
      let isMine = random(1) < 0.1;  // 10% chance of mine
      grid[c][r] = {
        x: c * cellSize,
        y: r * cellSize,
        mine: isMine,
        revealed: false,
        flagged: false
      };
    }
  }
}

function draw() {
  for (let c = 0; c < cols; c++) {
    for (let r = 0; r < rows; r++) {
      let cell = grid[c][r];
      stroke(0);
      if (cell.revealed) {
        fill(cell.mine ? color(255, 0, 0) : color(200));
      } else if (cell.flagged) {
        fill(255, 200, 0);
      } else {
        fill(100);
      }
      rect(cell.x, cell.y, cellSize, cellSize);
    }
  }
}

function mousePressed() {
  let c = floor(mouseX / cellSize);
  let r = floor(mouseY / cellSize);
  if (c >= 0 && c < cols && r >= 0 && r < rows) {
    let cell = grid[c][r];
    if (mouseButton === RIGHT) {
      if (!cell.revealed) cell.flagged = !cell.flagged;
    } else if (mouseButton === LEFT) {
      if (!cell.flagged) cell.revealed = true;
    }
  }
}

Game Tutorial — Platformer with Physics

This tutorial walked me through building a side-scrolling platformer from scratch using p5.js. I learned how to implement gravity by incrementing vertical velocity each frame, how to detect and resolve collisions between the player rectangle and platform rectangles, and how to use objects to store all the state for a player (x, y, vx, vy, isGrounded). I also added coin collection with a score counter, sound effects triggered on pickup, and randomly generated platforms for replayability. The biggest challenge was the collision resolution — snapping the player to the top of a platform only when falling downward, so that jumping through platforms from below still works correctly.

// ---- Platformer Tutorial — Physics, Collisions, Coins ----

let player = {
  x: 50, y: 50, w: 30, h: 30,
  vx: 0, vy: 0, speed: 5,
  jumpPower: -12, isGrounded: false
};

let platforms = [];
let coins = [];
let score = 0;
let gravity = 0.6;

function setup() {
  createCanvas(600, 400);

  // Generate random platforms
  for (let i = 0; i < 5; i++) {
    platforms.push({
      x: random(0, width - 100),
      y: random(100, height - 80),
      w: random(80, 150),
      h: 20
    });
  }
  platforms.push({ x: 0, y: height - 20, w: width, h: 20 }); // floor

  // Generate random coins
  for (let i = 0; i < 8; i++) {
    coins.push({
      x: random(50, width - 50),
      y: random(50, height - 100),
      size: 25,
      collected: false
    });
  }
}

function handlePlayerInput() {
  player.vx = 0;
  if (keyIsDown(LEFT_ARROW))  player.vx = -player.speed;
  if (keyIsDown(RIGHT_ARROW)) player.vx =  player.speed;
  if (keyIsDown(UP_ARROW) && player.isGrounded) {
    player.vy = player.jumpPower;
    player.isGrounded = false;
  }
}

function applyPhysics() {
  player.vy += gravity;
  player.x  += player.vx;
  player.y  += player.vy;
}

function checkPlatformCollisions() {
  player.isGrounded = false;
  for (let p of platforms) {
    if (player.x + player.w > p.x && player.x < p.x + p.w &&
        player.y + player.h > p.y && player.y < p.y + p.h) {
      if (player.vy > 0) {          // only snap when falling down
        player.y = p.y - player.h;
        player.vy = 0;
        player.isGrounded = true;
      }
    }
  }
}

function checkCoinCollections() {
  for (let c of coins) {
    if (!c.collected) {
      let d = dist(player.x + player.w/2, player.y + player.h/2, c.x, c.y);
      if (d < player.w/2 + c.size/2) {
        c.collected = true;
        score += 10;
      }
    }
  }
}

function draw() {
  background(30);

  handlePlayerInput();
  applyPhysics();
  checkPlatformCollisions();
  checkCoinCollections();

  // Draw player
  fill(255, 200, 0);
  rect(player.x, player.y, player.w, player.h);

  // Draw platforms
  fill(100);
  noStroke();
  for (let p of platforms) rect(p.x, p.y, p.w, p.h);

  // Draw coins
  fill(255, 215, 0);
  for (let c of coins) {
    if (!c.collected) ellipse(c.x, c.y, c.size);
  }

  // Score display
  fill(255);
  textSize(20);
  textAlign(LEFT, TOP);
  text("Score: " + score, 10, 10);
}

Final p5.js Project — Duck BBreaker

Duck BBreaker is my original culminating game built with p5.js. The player controls a duck horizontally at the bottom of the screen and launches a ring upward to smash rows of bricks — a take on the classic Breakout genre. I designed the duck and ring sprites myself in Adobe Illustrator and wrote the gameplay logic from scratch, using Gemini AI to help debug the boundary reflection math and get the initial file structure set up. The game features a title screen, speed settings (Slow / Medium / Fast), a lives system, a high-score tracker, and a game-over screen. Each game loop version (V1 → V3) added more polish: V1 was the basic collision system, V2 added the HUD and lives, and V3 added speed settings and responsive canvas sizing. The biggest design challenge was tuning the duck speed, ring speed, and brick layout so the game is actually beatable at Medium difficulty.
